# 微信云开发部署教程（新手友好版）

> 目标：让记录数据存储在微信云端，不同设备打开小程序都能看到同一份数据。
> 全程 GUI 操作，**不需要写代码**。

---

## 你需要准备什么
- 一个微信账号
- 5 分钟时间
- 不需要信用卡，云开发有免费额度，你的小程序完全够用

---

## 第一步：注册真实小程序 AppID（如果还没有）

你当前的 `project.config.json` 里是 `"touristappid"`，只能本地调试。必须换成真实的。

1. 打开浏览器，访问 **[微信公众平台](https://mp.weixin.qq.com/)**
2. 点右上角 **「立即注册」** → 选择 **「小程序」**
3. 按提示填写邮箱、密码，激活账号
4. 选择主体类型：**个人** 即可（不需要企业资质，也不需要交钱）
5. 填写个人信息，完成注册
6. 登录后，左侧菜单 → **「开发」** → **「开发管理」** → **「开发设置」**
7. 找到 **AppID**（以 `wx` 开头的一串字符），**复制下来**

> 如果已经有 AppID，直接跳到下一步。

---

## 第二步：在开发者工具中开启云开发

### 2.1 更换 AppID

1. 用微信开发者工具打开你的项目
2. 编辑 `project.config.json`，把 `"appid"` 从 `"touristappid"` 改成你的真实 AppID：

```json
"appid": "wx1234567890abcdef"
```

3. **保存文件**，开发者工具会提示重新登录/授权，确认即可

### 2.2 开通云开发

1. 在开发者工具顶部工具栏，点击 **「云开发」** 图标（一朵云 ☁️ 的图标）
2. 首次使用会弹出开通界面，点击 **「开通」**
3. 创建环境：
   - 环境名称：随便填，比如 `potty-dev`
   - **务必选择「免费版/体验版」**（有免费额度，够你用了）
   - 点击「确定」
4. 等待几秒，环境创建完成
5. 记下 **环境 ID**（格式类似 `cloud1-xxxxxx`），在云开发控制台左上角能看到

---

## 第三步：创建数据库集合

1. 在微信开发者工具中，点 **「云开发」** 进入云开发控制台
2. 左侧菜单 → **「数据库」**
3. 点击 **「添加集合」** 按钮
4. 集合名称输入：**`potty_records`**（必须写对这个名字，不能多也不能少）
5. 点击「确定」

### 设置集合权限：

6. 找到刚创建的 `potty_records` 集合，点击它
7. 顶部切换到 **「权限设置」** 标签
8. 选择 **「所有用户可读，仅创建者可写」**
9. 点击「保存」

---

## 第四步：修改项目配置文件

编辑项目根目录的 `config.js`，改两处：

### 修改 1：开启云模式

找到第 27 行：
```js
const USE_CLOUD = false;
```
改成：
```js
const USE_CLOUD = true;
```

### 修改 2：填写环境 ID

找到第 29 行：
```js
ENV: '',
```
在引号里填上你的**环境 ID**，比如：
```js
ENV: 'cloud1-xxxxxx',
```

改完后保存，这两行最终长这样：
```js
const USE_CLOUD = true;
const CLOUD = {
  ENV: 'cloud1-xxxxxx',    // ← 换成你自己的环境 ID
  COLLECTION_RECORDS: 'potty_records',
};
```

---

## 第五步：编译测试

1. 在微信开发者工具中点击 **「编译」**
2. 检查是否正常运行（没有报红字错误）
3. 试着记录一条数据，然后去云开发控制台 → 数据库 → `potty_records` 查看，应该能看到刚添加的记录

---

## 第六步：上传体验版测试

按照 `DEPLOY.md` 里的流程：

1. 开发者工具右上角点 **「上传」**，填版本号 `1.0.0`
2. 登录 [mp.weixin.qq.com](https://mp.weixin.qq.com/) → 管理 → 版本管理
3. 找到上传的版本 → 点 **「选为体验版」**
4. 管理 → 成员管理 → 体验成员 → 添加体验者（输入对方微信号）
5. 把体验版二维码发给对方

现在两台手机扫同一个码，记录的数据就能互相看到了 🎉

---

## 验证是否成功

最简单的验证方法：

1. **手机 A**：打开体验版，记录一条「小便」
2. **手机 B**：打开同一个体验版，切到「历史」页面

如果手机 B 能看到手机 A 刚记的那条记录，就说明**云端同步成功了**。

## 常见问题

### Q: 编译报错「未找到 cloud 对象」
A: 检查第二步，确保在开发者工具中成功开通了云开发，且 `config.js` 里的 `ENV` 和开发者工具里的环境 ID 完全一致。

### Q: 记录数据后在控制台看不到
A: 检查第三步，集合名必须是 `potty_records`，权限必须是「所有用户可读，仅创建者可写」。

### Q: 两台手机数据不同步
A: 确认两台手机扫码的是**同一个体验版**（同一个 AppID + 同一环境 ID）。下拉刷新历史页看看。

### Q: 之前本地存储的数据怎么办？
A: 目前本地数据和云端数据是独立的。建议先本地手动记录一遍到云端（数据量小很快），或者我可以给你加一个「导入本地数据到云端」的功能。

## 改了什么文件

| 文件 | 改动 | 你手动？ |
|------|------|----------|
| `project.config.json` | `appid` 换真实 AppID | ✅ 手动 |
| `config.js` | `USE_CLOUD = true` + 填 `ENV` | ✅ 手动 |
| `potty_records` 集合 | 云数据库里新建 | ✅ 手动 |
| `app.js` / `utils/cloud.js` / `store.js` | 云开发逻辑（已写好） | ❌ 不需要动 |
